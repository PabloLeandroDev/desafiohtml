Variáveis e suas Funcionalidades

Ola desenvolvedor(a), neste site iremos passar informações sobre a linguagem de programação pwn que atualmente e utilizada no SAMP (San Andreas Multiplayer).
Nosso objetivo é ajudar você no processo de apredizado da linguagem, apoiando com informações e tutoriais na sequência correta para aprender a desenvolver sistemas no samp.

Conceito:

Uma variável tem uma função de armazenar um dado que poderá ser utilizado eventualmente, você poderá determinar em uma variável o seu respectivo nome, tipo de dado e posição no código (local ou global).
Para criar uma variável dentro da linguagem pawn você deverá utilizar a reservada palavra new

Exemplo:

new MinhaVariavel;

Tipos de Variáveis:

Existe alguns tipos de variáveis dentro da linguagem pawn, você precisará definir o tipo de variável de acordo com o dado que deseja armazenar.
Os tipos são int: "valor inteiro '5'", float: "valor decimal 8.3452", bool: "valor boelano verdadeiro/false".

Por padrão sempre que uma variável por criada, ela terá seu tipo definido como int: inteiro.

Exemplos de Criação:

new valorInt - Perceba como foi citado acima, por não definir um valor no inicio já será derterminada como valor interior.
new float:Posicao - Aqui será uma variável definida para guardar valores decimais.
new bool:Condicao - Aqui será uma variável definida para guardar um valor verdeiro ou falso.

Variaveis Globais:

Ao contrário da variável local, a global poderá ser utilizada em qualquer parte de seu código do início ao fim.É sempre importante se perguntar antes se realmente será necessário a criação de uma variável global,
pois ela ficará ocupando espaço da memória do seu servidor enquanto ele estiver ligado. Para criar uma variável global, basta apenas adicionar a sintaxe abaixo fora qualquer bloco de comando.

Exemplo:

new MinhaVariavelGlobal;

Variaveis Locais:

A variável local é usada apenas em uma determinada parte do código, sendo assim depois de executada ela será apagada da memória do servidor.
É importante sempre tentar utilizar variáveis locais em seu projeto, pois evitará ocupar espaço de memória do seu servidor.

Exemplo:

public OnPlayerConnect(playerid)
{
 new varLocal;
 return 1;
}

No exemplo acima, perceba que a variável "varLocal" está dentro da CallBack "OnPlayerConnect", sendo assim apenas funcionará quando um jogador conectar e logo em seguida apagará.
